Engineering Manager Take-Home Assignment - Java

 

 

Problem trying to solve:

“We have the ambitious goal of rebuilding Amazon.com’s shopping experience as a set of micro-services. To keep the scope narrow, please focus on the core features that relate to the “core user activity” of “purchasing an item”. The mandatory services to include in the design are the authentication service and the item search service.“ – Disqo 

 

Thinking Process Given Short Description of the Requirements:

1.     Experience user flow from Amazon.com and build a mental model of the product

a.     Let’s go to Amazon.com and play around the interface, search for products, add products to cart, open an account

b.     Document basic user flow:

                                               i.     User views the homepage

                                             ii.     User enters “Shoes” in the search box

                                            iii.     User clicks Search Icon

                                            iv.     User views product search results

                                              v.     User clicks on a product

                                            vi.     User clicks on Add to Cart

                                           vii.     User clicks Checkout

                                         viii.     User views the login page

                                            ix.     User logs in with username and password

                                              x.     User enters shipping and billing information

                                            xi.     User views order confirmation page

                                           xii.     User gets an e-mail order confirmation

2.     Given the user flows & requirements build a technical specification on what is needed to accomplish this task

a.     High level of needed components

                                               i.     Storefront UI

                                             ii.     Product inventory and search capability

                                            iii.     User and billing management

b.     Given the main base components for a base commerce site, let’s dedicate micro-services to accomplishing the task, please see Micro-Services, Framework & API Definitions section

KPI Definitions

 

We will be defining KPIs based on the versioning and user acceptance criteria. There are 4 minor and major releases development team will be doing. We will be tying each version with a goal that we want to achieve incrementally.

 

·      Version 0.1

o   User acceptance:

§  Test Driven Development: 5 passing unit tests defined for each micro-service.

§  Storefront user interface displays at least 1 product item accessible through internal domain. Doesn’t need to be nicely designed

§  At least 1 commit for each codebase defined

·      Version 0.5

o   User acceptance:

§  User can view the storefront with 50 product items from different categories

§  Site load is under 5 seconds

§  Google Search Crawler can access and crawl the publicly available resources

§  Logging availability in a DataDog Dashboard.

·      Version 0.9

o   User acceptance:

§  External user can access the site through a secure publicly available domain.

§  Site and DBs passed the stress test criteria. Micro-services and DBs can process 4000 requests per second.

§  Site has 100 internal users defined

§  Engineering dashboard displaying each micro-service’s CPU and memory utilization in DataDog

·      Version 1.0

o   User acceptance:

§  All publicly available pages indexed on Google and Bing Search engines

§  Site passes McAfee penetration tests

§  Site processed 10 purchases. Google Analytics can showcase flow of purchase

§  Site is advertised on Google Adwords and receives at least 1 purchase within the first week

 

Note to self: After the release of Version 1.0, it is highly recommended to do a team outing and celebrate.

Responsibilities:

         Authentication: Responsible for authenticating and listing the user. DB Schema:

Storefront UI: Responsible for serving the user content

Product Inventory: Responsible for keeping inventory of catalog items

Pricing: Responsible for providing the pricing of a product

Orders: Responsible for registering a user order and processing it

Shipping Calculator: Responsible for providing user with shipping information based on their location and items

Billing: Responsible for billing the user’s payment
